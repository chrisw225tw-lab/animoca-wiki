# Tokenomics

**Tokenomics**（代幣經濟學）是設計和管理加密代幣經濟系統的學科。[[Animoca-Brands]] 在這方面具有豐富經驗，設計了多個成功的代幣模型。

---

## Animoca 的代幣設計哲學

### 核心原則
| 原則 | 說明 |
|------|------|
| 實用性優先 | 代幣必須有明確的使用場景 |
| 可持續性 | 經濟模型能長期運作 |
| 社群所有 | 讓用戶分享平台價值增長 |
| 簡單透明 | 規則清晰，易於理解 |

### 設計目標
- **激勵對齊**：用戶、投資者、團隊利益一致
- **價值累積**：代幣捕捉平台創造的價值
- 網絡效應**：更多人使用提升代幣價值

---

## 關鍵設計要素

### 1. 代幣用途 (Utility)
成功代幣需要多種使用場景：
- **支付功能**：購買商品和服務
- **治理權利**：參與協議決策
- **質押獎勵**：鎖定代幣獲得收益
- **訪問權限**：解鎖高級功能

### 2. 供應管理
| 要素 | 考量 |
|------|------|
| 總供應量 | 稀缺性與實用性的平衡 |
| 發行速度 | 控制通脹率 |
| 銷毀機制 | 創造通縮壓力 |
| 解鎖時間表 | 防止供應衝擊 |

### 3. 需求驅動
創造持續的代幣需求：
- **必需性**：完成某些操作必須使用代幣
- **折扣激勵**：使用代幣獲得優惠
- **質押收益**：鎖定代幣的被動收入
- **投機需求**：對未來價值的預期

## 常見代幣模型

### 治理代幣
- 用於協議決策投票
- 例子：UNI、COMP、AAVE
- 風險：價值捕獲不明確

### 實用代幣
- 必須用於平台服務
- 例子：SAND、MANA、AXS
- 優勢：需求與平台使用掛鉤

### 雙代幣系統
- 一個治理 + 一個實用/遊戲內貨幣
- 例子：AXS + SLP、SAND + GEM
- 優勢：分離投資和消費功能

## 最佳實踐與陷阱

### 應該做的
✅ 代幣有明確的必需用途
✅ 長期的解鎖時間表
✅ 真正的去中心化治理
✅ 透明的經濟數據

### 應該避免的
❌ 純粹的炒作型代幣
❌ 過短的鎖倉期
❌ 過度複雜的機制
❌ 團隊控制過多供應
